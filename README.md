I'm a cybersecurity engineer. I've spent 6 years in SOC and Blue Team work — the part of security that watches, detects, and responds rather than builds walls.

Right now I work at **epay (Euronet)** in Munich. Before that, Stockholm. Before that, Mumbai. The geography changed; the craft didn't.

What I find genuinely interesting about this field: you're always one step behind the attacker, and that asymmetry is what makes it intellectually serious. You can't brute-force your way to good detection. You have to *think* like the thing you're hunting.

---

## Things I know well

**On the defensive side —** I've worked extensively with Cortex XSOAR, ArcSight, and log ingestion pipelines. I write SOAR playbooks that turn 20-minute manual triage into 90-second automated response. I've built Threat Intelligence Platforms from scratch, integrated TAXII feeds, and run adversarial phishing campaigns to test whether security training actually sticks.

**On the building side —** Python is my first language for security tooling. Django and Flask for web, FastAPI for APIs. I also know my way around Flutter and React when I'm building something cross-platform. I care about UX — I did an MSc in it at Jönköping University, and it changed how I think about every tool I build or configure.

**Something I believe:** most security failures aren't technical. They're human. That's why I spent a year studying how people think and interact with interfaces. A security tool that nobody uses correctly isn't a security tool.

## What I've worked on

**Reliance Industries, Mumbai** — my first three years in the field. Built a Threat Intelligence Platform from zero using open-source tooling and TAXII feeds. Designed an automated penetration testing system with modules for SQLi, XSS, SSL vulnerabilities, and firewall alertness. Built a CTF platform the company used for security hiring. Ran company-wide phishing simulations with a scoring system to find the gaps.

**Aqilea (formerly Soltia), Stockholm** — client-side Blue Team work for a major telecom. Ran Cortex XSOAR multi-tenancy for a large, complex environment. Optimised log ingestion pipelines. Reduced incident resolution times measurably. Migrated from expensive enterprise tooling to leaner open-source alternatives without losing capability.

**epay, Munich** — current. IT Security DevOps in a payments infrastructure context. Still learning the domain. Enjoying the complexity.

## A few things I find interesting

- The cat-and-mouse nature of threat intelligence. Feeds go stale. Actors change TTPs. You're always updating your mental model.
- The moment an analyst instinct — "something's off" — resolves into a confirmed incident. That gap between signal and certainty.
- Why most organisations are breached through phishing, not zero-days. The human surface area is enormous and hard to patch.
- Building tools that adapt to the user instead of forcing the user to adapt.
